レスポンシブ

レスポンシブとは、サーバーからどのデバイスに対しても常に同じHTMLコードを配信しつつ、 CSSを使用して各デバイスでのページの表示の仕方を変更する設定方法です。 現在ではWebサイトを閲覧する際に使用するデバイスとして、PCよりもスマートフォンなどの画面が 縦長のものであることが多いです。したがって「ユーザにとって見やすくいコンテンツ」と言うときの 「ユーザ」の多くが「スマートフォンを通してサイトを見るユーザ」と考えられます。したがって、 Webサイトを作る本来の目的が多くの人に閲覧してもらうことを前提にしている場合、モバイル向けの デザインを実装することは必須です。

モバイル向けのデザインとしてどのようなものが適切であるかは他の「デザイン」や「レイアウト」の ページで学習してもらうこととして、ここではどのように場合分けをするかに着目して解説します。

レスポンシブデザインのメリット・デメリット

メリット

  • URLが1つで済み、ユーザによるコンテンツの共有やリンクが容易になる。 （レスポンシブデザインにしない場合、同じ内容を複数のページで記述する必要があるため）
  • URLが1つで済み、ページ管理者にとって、維持管理が比較的容易となる。
  • Googlebotがサイトをクロールするために必要なリソースを節約できる。（つまり人間が 閲覧するときも1つのサイトで様々なデバイスに対応しているために見やすい）

デメリット

  • PC向けのCSSもモバイルで閲覧しているときに読み込むことになり、サイト表示に時間がかかる。
  • PCとモバイル向けでデザインを大幅に変更するのは困難。（HTMLは同じであるため）
  • デバイスごとにデザインを制作する必要があり、制作のための時間や費用のコストが大きくなる。

実装方法

HTML内の記述

HTMLのhead要素内に次のように記述します。 

<meta name="vieport" content="width=device-width, initial-scale=1">

CSS内の記述

レスポンシブデザインでは、メディアクエリという機能を用いて1つのページで異なるスタイルをしてします。 メディアクエリはCSSの仕様の1つで、表示された画面環境に応じて適用するスタイルを切り替える機能のことを言います。 メディアクエリの書式にはいくつかパターンがありますが、ここでは必要最低限の書式と例を提示します。

まず、代表的な書式を示します。 

@media (条件) {
    /* 条件にあった場合に適用される CSS */
}

実際は(条件)の前にメディアのタイプなども必要ですが、「全て」の場合には省略可です。 必要に応じて付け加えてください。ここでの説明では不要です。

次に、場合分けを行った例を示します。多くのサイトの場合、スマートフォンなどの縦長で小さい 画面向けの方がデザインが簡素化される傾向にあり、多くの場合、小さい画面を優先的にデフォルトにします。 

@charset "utf-8";
/*デフォルト（480px 未満）
  デバイスで共通で利用するスタイル
  スマートフォンに最適化したレイアウト
  */
@media screen and (min-width : 480px){
  */ 最小幅 480px 以上の指定 */
}
@media screen and (min-width : 768px){
  */ 最小幅 768px 以上の指定 */
}
@media screen and (min-width : 1024px){
  */ 最小幅 1024px 以上の指定 */
}

まとめ

このようにして場合分けを行い、CSSを記述することができます。あとはそれぞれに適切なデザインを組み合わせればよいです。 ここで注意すべきはモバイル用に作られたメディアクエリなどですが、それはもう与えられたものとして我々が使うことができる ということです。つまり、この機能を見た皆さんは、スクリーンのサイズによってCSSを場合分けできるという技術を 何かほかのことに使える可能性もあります。想像的に新たなコンテンツを作成してください。
